## Summary

This PR adds aqlprofile test infrastructure to TheRock, replacing the
previously reverted PR #2055. The key CMake change switches aqlprofile from
`DISABLE_AMDGPU_TARGETS` to `USE_DIST_AMDGPU_TARGETS`, enabling proper
per-target builds (resolving #2704). It adds a new `aqlprofile-tests` artifact,
a CI test configuration, a test runner script, and the `--aqlprofile` flag to
`install_rocm_from_artifacts.py`.

**Net changes:** +81 lines, -4 lines across 7 files

## Detailed Review

### 1. `profiler/CMakeLists.txt`

#### ⚠️ IMPORTANT: CMake indentation — `if(THEROCK_BUILD_TESTING)` block not indented

Lines 53–61 of the merged file:
```cmake
if(THEROCK_BUILD_TESTING)
  therock_provide_artifact(aqlprofile-tests
    DESCRIPTOR artifact-aqlprofile.toml
    ...
  )
endif(THEROCK_BUILD_TESTING)
```

This block lives inside `if(THEROCK_ENABLE_ROCPROFV3)` (line 4), but is not
indented to match. Every other block inside that outer `if` uses 2-space
indentation. Compare with e.g. `therock_provide_artifact(rocprofiler-sdk ...)`
at line 194 which is correctly indented.

**Recommendation:** Indent the block 2 spaces to match the surrounding code.

#### 💡 SUGGESTION: CMake change from `DISABLE_AMDGPU_TARGETS` to `USE_DIST_AMDGPU_TARGETS`

The original code had:
```cmake
DISABLE_AMDGPU_TARGETS
```
with a TODO comment referencing https://github.com/ROCm/aqlprofile/issues/2.

The PR replaces this with `USE_DIST_AMDGPU_TARGETS` and adds HIP compiler
toolchain + dependencies (googletest, hip-clr). This is the correct fix — the
upstream issue was resolved and the tests can now build for explicit GPU
targets. Good change.

### 2. `build_tools/github_actions/test_executable_scripts/test_aqlprofile.py`

#### ⚠️ IMPORTANT: `LD_LIBRARY_PATH` set to Path object, not string

Line 17:
```python
env["LD_LIBRARY_PATH"] = THEROCK_DIR / "lib"
```

This assigns a `pathlib.Path` to an env dict value. While Python's subprocess
can handle this via `os.fsencode()`, the established pattern (see
`test_rocprofiler_compute.py:23`, `test_rocprofiler_compute.py:37-39`) is to
use `str()`:
```python
environ_vars["ROCM_PATH"] = str(THEROCK_PATH)
```

**Recommendation:** Use `str(THEROCK_DIR / "lib")` for consistency.

#### ⚠️ IMPORTANT: Existing `LD_LIBRARY_PATH` is clobbered

The assignment overwrites any existing `LD_LIBRARY_PATH` rather than prepending.
Compare with `test_rocprofiler_compute.py` (lines 34-41) which preserves the
existing value:
```python
old_ld_lib_path = os.getenv("LD_LIBRARY_PATH", "")
if old_ld_lib_path:
    environ_vars["LD_LIBRARY_PATH"] = f"{THEROCK_LIB_PATH}:{old_ld_lib_path}"
```

This may work in CI where `LD_LIBRARY_PATH` is typically empty, but it
breaks the established pattern and could cause issues in other environments.

**Recommendation:** Prepend to existing `LD_LIBRARY_PATH` like other test scripts.

#### 💡 SUGGESTION: Unused `SCRIPT_DIR` variable

`SCRIPT_DIR` (line 13) is defined but never referenced. Other scripts use it
to resolve the repo root (`THEROCK_DIR = SCRIPT_DIR.parent.parent.parent`).

#### 💡 SUGGESTION: No null guard on `OUTPUT_ARTIFACTS_DIR`

Line 14:
```python
THEROCK_DIR = Path(os.getenv("OUTPUT_ARTIFACTS_DIR")).resolve()
```

If `OUTPUT_ARTIFACTS_DIR` is unset, `os.getenv()` returns `None` and
`Path(None)` raises `TypeError`. Other scripts use the same pattern, so this
is consistent, but a guard or descriptive error would be friendlier.

#### ❌ BLOCKING (already fixed): Missing SPDX copyright header

The diff shows the file was created without the standard copyright header:
```python
# Copyright Advanced Micro Devices, Inc.
# SPDX-License-Identifier: MIT
```

This has been fixed in the current `main` (the file now has the header).

### 3. `build_tools/github_actions/fetch_test_configurations.py`

#### ❌ BLOCKING (already fixed): `total_shards` instead of `total_shards_dict`

The diff shows:
```python
"total_shards": 1,
```

Every other entry uses `total_shards_dict`:
```python
"total_shards_dict": {
    "linux": 1,
    "windows": 1,
},
```

The processing code (line 510) does:
```python
total_shards = job_config_data.get("total_shards_dict", {}).get(platform, 1)
```

So `total_shards: 1` is silently ignored and defaults to 1. This happened to
work because the default is 1 and aqlprofile only needs 1 shard, but it's a
latent inconsistency. This has been fixed in the current `main`.

### 4. `BUILD_TOPOLOGY.toml`

The new entry looks correct:
```toml
[artifacts.aqlprofile-tests]
artifact_group = "profiler-core"
type = "target-specific"
artifact_deps = ["core-runtime"]
feature_group = "PROFILER"
disable_platforms = ["windows"]
```

- `type = "target-specific"` is correct since aqlprofile builds per-GPU-target
- `artifact_deps = ["core-runtime"]` is appropriate — tests need the runtime
- `feature_group = "PROFILER"` and `disable_platforms = ["windows"]` match
  other profiler entries
- No `feature_name` is needed — auto-generation from the artifact name is the
  common pattern for target-specific artifacts

### 5. `build_tools/install_rocm_from_artifacts.py`

The `--aqlprofile` flag is added correctly following the existing pattern.
When `--aqlprofile --tests` is passed, the flow is:
1. `args.aqlprofile` → `extra_artifacts.append("aqlprofile-tests")`
2. `args.tests` → pattern becomes `aqlprofile-tests_test`

This works but the naming is mildly confusing: `--aqlprofile` maps to
`aqlprofile-tests` (not the base `aqlprofile` library, which is bundled in
`rocprofiler-sdk`). The test config's `fetch_artifact_args` correctly
passes `"--aqlprofile --tests"`.

### 6. `profiler/artifact-aqlprofile.toml`

#### ❌ BLOCKING: Artifact descriptor duplicates files already in `rocprofiler-sdk`

The new `artifact-aqlprofile.toml` is a **verbatim copy** of the aqlprofile
section already present in `artifact-rocprofiler-sdk.toml` (lines 1–10):

```toml
# Both files contain identical content:
[components.dbg."profiler/aqlprofile/stage"]
[components.dev."profiler/aqlprofile/stage"]
[components.doc."profiler/aqlprofile/stage"]
[components.lib."profiler/aqlprofile/stage"]
[components.run."profiler/aqlprofile/stage"]
[components.test."profiler/aqlprofile/stage"]
include = [
  "share/hsa-amd-aqlprofile/**",
]
```

This creates two problems:

1. **Unnecessary components:** The CMake `therock_provide_artifact(aqlprofile-tests)`
   only selects `COMPONENTS test`, so the dbg/dev/doc/lib/run component
   definitions in `artifact-aqlprofile.toml` are dead weight — they're never
   activated by this artifact.

2. **Duplicate file ownership:** The `test` component with the
   `share/hsa-amd-aqlprofile/**` include pattern is claimed by both the
   `rocprofiler-sdk` artifact (via `artifact-rocprofiler-sdk.toml` line 7–10)
   and the `aqlprofile-tests` artifact (via `artifact-aqlprofile.toml`
   line 7–10). The same files from `profiler/aqlprofile/stage` get packaged
   into both artifacts.

**Required action:** The `artifact-aqlprofile.toml` should only define the
`test` component. And the aqlprofile test component should be removed from
`artifact-rocprofiler-sdk.toml` since test files now have their own dedicated
artifact. The cleaned-up `artifact-aqlprofile.toml` should be:
```toml
# aqlprofile tests
[components.test."profiler/aqlprofile/stage"]
include = [
  "share/hsa-amd-aqlprofile/**",
]
```

And `artifact-rocprofiler-sdk.toml` lines 7–10 should either drop the test
component for aqlprofile or remove its `include` pattern so only
`aqlprofile-tests` owns those test files.

### 7. `docs/development/windows_support.md`

Correctly adds the aqlprofile entry to the windows support table as
unsupported, consistent with `disable_platforms = ["windows"]`.
